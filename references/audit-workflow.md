# CRO 审计工作流

系统化审计 Shopify 站点，按优先级找出影响转化率的问题。

---

## 审计阶段

### Phase 1 — 速度审计（P0 阻塞性）

**工具：** Chrome DevTools Lighthouse（Mobile 模式，Slow 4G 模拟）

| 指标 | 目标 | 不达标影响 |
|------|------|-----------|
| LCP | < 2.5s | 首屏过慢 → 跳出 |
| CLS | < 0.1 | 布局跳动 → 误触/放弃 |
| TBT | < 200ms | 交互卡顿 → 放弃 |
| 页面总重 | < 2MB | 慢速网络无法加载 |
| HTTP 请求 | < 50 | 过多请求 → 加载慢 |

**执行步骤：**
1. 打开 Chrome DevTools → Lighthouse → 选中 Performance + Accessibility
2. 选择 Mobile 设备 + Slow 4G 模拟
3. 运行审计
4. 标记所有 ≥ 200KB 的图片
5. 标记所有 Render-blocking resources
6. 在 Shopify Admin → Online Store → Speed 查看 Shopify 速度分
7. 检查 `theme.liquid` 中 App 注入的 `<script>` 和 `<style>`

**常见阻塞性问题修复：**
- 图片未使用 `| image_url: width: N` → 全部替换
- 第三方脚本同步加载 → 添加 `defer` 或 `async`
- 未使用的 App 残留代码 → 删除
- Google Fonts 无预连接 → 添加 `<link rel="preconnect">`

### Phase 2 — 首页审计（P0 阻塞性）

逐项检查：

1. **首屏 3 秒测试：** 打开页面 3 秒后关闭，能不能说出「卖什么」+「为什么买你」？
2. **CTA 检查：** 首屏是否有且只有一个醒目的按钮？按钮颜色是否与背景对比度 ≥ 4.5:1？
3. **信任信号：** 首屏可见范围内是否有评价数、媒体 Logo、免邮、退换政策等信任要素？
4. **移动端溢出：** 375px 宽度下是否有横向滚动条？Chrome DevTools → 设备模拟 → 375px
5. **导航项数：** 主导航是否 ≤ 7 项？多余项是否可合并或移到 footer？

### Phase 3 — PDP 审计（P1 高优先）

1. **产品图片：** 是否 ≥ 5 张？是否支持缩放？首张是否优先加载？
2. **变体选择器：** 颜色变体是否用色块（非下拉）？尺寸变体是否用按钮？缺货是否可见？
3. **加购按钮：** 是否醒目？点击后是否有反馈（抽屉弹出 / 角标更新）？移动端是否 Sticky？
4. **库存状态：** 是否有库存/缺货明确标识？是真实数据吗？
5. **价格信息：** 是否含税费/运费说明？是否有对比原价和折扣百分比？
6. **产品描述：** 是否用 bullet points？是否用折叠面板组织？
7. **信任加速器：** 加购按钮下方是否有信任信号？

### Phase 4 — 购物车/结账审计（P1 高优先）

1. **加购反馈：** 是否明确（非静默）？
2. **数量修改：** 购物车内是否可修改数量？
3. **运费展示：** 是否尽早显示（不等结账最后一步）？
4. **折扣码入口：** 是否不被强调（避免用户出去找折扣码）？
5. **免邮提示：** 是否有免邮门槛进度条？

## 全站通用检查项

- [ ] Logo 点击返回首页
- [ ] 404 页面有搜索栏 + 推荐产品 + 返回首页链接
- [ ] 多币种自动切换（Shopify Markets 检查）
- [ ] 全站 HTTPS，无混合内容
- [ ] 每页只有一个 `h1`
- [ ] 所有 `<img>` 有 `alt` 属性
- [ ] 所有 `<button>` 有文本或 `aria-label`
- [ ] 支付方式图标在 footer 和购物车中可见

---

## 常见反模式与修复

| 反模式 | 后果 | 修复方案 |
|--------|------|----------|
| 首页塞满所有产品 | 用户决策瘫痪 | 精选 3-6 个明星产品，其余放入合集页 |
| 弹窗太多太早 | 跳出率飙升 | 弹窗延迟 ≥ 15 秒，或用退出意图触发（`mouseleave document`） |
| 轮播图超过 2 个 | 速度变慢，分散注意力 | 1 个 Hero 轮播 + 1 个静态内容区 |
| 产品图不统一 | 显得不专业 | 统一背景、比例、光影方向 |
| 字体太小/太灰 | 可读性差 | 正文 ≥ 14px (移动 ≥ 16px)，对比度 ≥ 4.5:1 |
| 假冒的紧迫感 | 失去信任 | 倒计时结束真恢复原价；库存用 `inventory_quantity` |
| 忽视 404 页面 | 无效流量流失 | 添加搜索栏 + 推荐产品链接 + 返回首页 |
| 多币种无自动切换 | 国际用户流失 | Shopify Markets 自动检测 IP 切换币种 |
| 颜色作为唯一信息载体 | 无障碍问题 | 加文字或图标辅助 |

## 审计输出模板

```markdown
## 站点审计报告：{站点名}

### 总体评分
- 视觉设计：X/10
- 移动端体验：X/10
- 性能表现：X/10 (Shopify 速度分: X)
- 转化率潜力：X/10

### 关键问题（按优先级）
1. **[P0 - 阻塞性]** - 问题 + 修复方案 + 预计影响
2. **[P1 - 高优先]** - 问题 + 修复方案
3. **[P2 - 优化]** - 问题 + 修复建议

### 快速修复（无需开发资源）
- [ ] 任务 1
- [ ] 任务 2

### 需要开发资源的改进
1. 改进项 — 涉及文件 + 建议实现
```

## 反合理化陷阱

以下想法**错误**，Agent 不得采纳：

| 陷阱 | 正确认知 |
|------|----------|
| "这个改动太小，不用检查性能" | 每个图片、每个 script 累加，累计效应显著 |
| "移动端以后再说" | 70% 流量来自移动端。先桌面后移动 = 两倍工作量 |
| "加个轮播好看" | 轮播几乎不提升转化。用户只看第一张 |
| "加个弹窗收集邮箱" | 弹窗过多 = 跳出。延迟 ≥ 15 秒 |
| "用这个第三方插件很快" | 每个插件注入 JS/CSS = 增加加载时间 |
| "库存紧张写 3 件显紧迫感" | 用户能察觉造假。丢失的信任 > 一个订单 |
